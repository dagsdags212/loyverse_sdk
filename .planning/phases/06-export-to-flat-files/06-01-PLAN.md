<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller → exporter | User-provided filepath (untrusted — OS resolves) |
| Pydantic models → Polars | In-memory data already validated by Pydantic — no injection surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | `export_csv()` / `export_parquet()` filepath param | accept | File writes delegate to OS; Polars handles I/O. User controls filepath — standard OS permissions apply. No path traversal risk since user provides the path. |
| T-06-02 | Information Disclosure | Exported files | accept | Export files contain the same data already retrieved from the API. User controls output location. No additional exposure beyond what the API already provides. |
| T-06-03 | Denial of Service | Large dataset export | accept | No rate limiting or size checks. User controls input size. Polars handles memory management. Acceptable for a library SDK — caller is responsible for resource bounds. |
| T-06-SC | Tampering | Dependencies (polars, pyarrow) | mitigate | Both already in `uv.lock` with pinned versions. No new dependencies added in this phase. |
</threat_model>
