## Critical Gaps

### 🔴 BLOCKER: PaymentTypes endpoint missing from RESOURCE_ORDER consumer

**Evidence:** `payment_types` listed in `RESOURCE_ORDER` at `src/loyverse_sdk/db/exporter.py:54`. No `PaymentTypesEndpoint` exists — no endpoint file, no import in `endpoints/__init__.py`, no instantiation in `client.py`. Schema and converters exist (`PaymentTypeDB` class, `_split_payment_types()` in converters) but the endpoint to fetch data from the API does not exist.

**Impact:** DuckDB full export fails with `KeyError` at the `payment_types` step. Selective exports that skip `payment_types` work. The schema will have an empty `payment_types` table.

### 🟡 BLOCKER: Phase 03 — No Verification

Phase 03 is marked COMPLETE in the ROADMAP with 2 plans completed, 139 tests passing, and inventory fully integrated into the DuckDB export pipeline. However, no `*-VERIFICATION.md` file exists in the phase directory. The phase was never goal-verified.
