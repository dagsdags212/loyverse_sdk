## Critical Gaps

### 🔴 BLOCKER 1: Phase 04 Services Layer — All source files deleted

**Evidence:** Commits `9cf0b48`, `1f1c793`, `dbe8c7b`, `40e05b3` created service files; commit `e40c399` (`chore: removed services modules`) deleted the entire `src/loyverse_sdk/services/` directory; commit `ea5449a` removed service references from `client.py`; commit `1e764fd` deleted `tests/unit/services/`.

**Current state:** `src/loyverse_sdk/services/` does not exist. No imports of `loyverse_sdk.services` anywhere in the codebase. All 5 service classes (ItemsService, CustomersService, DiscountsService, CategoriesService, TaxesService) are inaccessible.

**Affected requirements:** SVC-ITEMS, SVC-CUSTOMERS, SVC-DISCOUNTS, SVC-CATEGORIES, SVC-TAXES — all unsatisfied.

**Impact:** Any consumer expecting to use the services layer will get `ModuleNotFoundError`. The phase VERIFICATION.md (which predates deletion) shows 4/5 truths verified but the code it verified no longer exists.

### 🔴 BLOCKER 2: PaymentTypes endpoint missing from RESOURCE_ORDER consumer

**Evidence:** `payment_types` listed in `RESOURCE_ORDER` at `src/loyverse_sdk/db/exporter.py:54`. No `PaymentTypesEndpoint` exists — no endpoint file, no import in `endpoints/__init__.py`, no instantiation in `client.py`. Schema and converters exist (`PaymentTypeDB` class, `_split_payment_types()` in converters) but the endpoint to fetch data from the API does not exist.

**Impact:** DuckDB full export fails with `KeyError` at the `payment_types` step. Selective exports that skip `payment_types` work. The schema will have an empty `payment_types` table.

### 🟡 BLOCKER 3: Phase 03 — No Verification

Phase 03 is marked COMPLETE in the ROADMAP with 2 plans completed, 139 tests passing, and inventory fully integrated into the DuckDB export pipeline. However, no `*-VERIFICATION.md` file exists in the phase directory. The phase was never goal-verified.

### 🟡 BLOCKER 4: Phase 05 — No Verification

Phase 05 is marked COMPLETE in the ROADMAP with all 123 tests passing and the full test suite green. However, no `*-VERIFICATION.md` file exists. The phase was never goal-verified.
